GitLab to GitHub Repository Importer

Posted on 05 February 2026

Welcome to GitLab to GitHub Repository Importer.

This tool was created to simplify the process of migrating repositories from GitLab to GitHub. Whether you need to import a single repository or perform bulk operations on an entire organization using manifest.xml files, this tool provides a streamlined solution.

With support for both anonymous and authenticated access, you can quickly migrate all branches and tags from your GitLab repositories to GitHub. The tool features automatic repository creation, preserves complete history, and respects GitHub API rate limits.

GitLab to GitHub Repository Importer is built with pure Python, requiring minimal dependencies. Simply run the script and follow the interactive prompts to start migrating your repositories.

Thank you for using GitLab to GitHub Repository Importer.

Features

  • Import multiple repositories from GitLab to GitHub
  • Support for manifest.xml parsing
  • Import to personal account or organization
  • Optional prefix for repository names
  • Custom GitLab base URL support
  • Automatic repository creation on GitHub
  • Mirror clone (all branches and tags)

Requirements

  • Python 3.6+
  • Git
  • requests library

Installation

pip install requests

Usage

Basic Usage

python gitlab_to_github_importer.py

Environment Variables

Set GitHub token as environment variable:

export GITHUB_TOKEN="your_github_token_here"

Interactive Mode

The script will prompt you for:

  • GitHub Personal Access Token (if not set in environment)
  • Path to manifest.xml file
  • Target (Personal Account or Organization)
  • Optional prefix for repository names
  • GitLab base URL (default: https://gitlab.com)

GitHub Token Permissions

Your GitHub Personal Access Token needs the following scopes:

  • repo (full control of private repositories)
  • admin:org (if importing to organization)

How It Works

  • Parses manifest.xml to extract repository information
  • Creates repositories on GitHub (personal or organization)
  • Clones repositories from GitLab using --mirror flag
  • Pushes all branches and tags to GitHub
  • Cleans up temporary files

Notes

  • Existing repositories on GitHub will be skipped with a warning
  • All branches and tags are imported
  • Repository history is preserved
  • Large repositories may take time to import
